# The brief

This is the second of three take home assignments related to building a backend web service to support a simple application called petR.

In the previous assignment you built the base for the petR RESTful API using Node.js and the Express framework. For this assignment you will enhance that base application to use MongoDB for data persistance and add a second user resource.

# Core Requirements

  1. Using the Express.js framework, the API will expose a full set of CRUD routes (six, including both put and patch) for each of two resources: pet, and user. All API resource paths must begin with /api.

  2. Resource collections will be stored in MongoDB and accessed via Mongoose Model classes. The Model.schema for the resource objects will have the following properties.

Remember

The MongoDB driver will automatically assign the _id property.

  1. The owner property of the Pet model should notbe an object id referencing the User model.

  2. Each resource should have its own Router module (opens new window).

  3. Routes related to individual members of a resource collection should return a properly formatted error response with an error object for any invalid req.body or not found req.params.id value.All errors messages should end with 2 periods

  4. Ensure that you write clean and readable code. Pay attention to:

  • no runtime errors
  • follow the project structure used in this course
  • properly formatted JSON responses with either data or error at the root
  • no debugging console logs
  • no commented out code
  • semantically descriptive names for variables and functions
  • use a .gitignore file and do not commit node_modules/ or .env files
  • include .env.example file
  • properly formatted package.json file
    • correct project name
    • your author details
    • start and dev scripts

# NOTE - Validation can now be taken care of by mongoose
