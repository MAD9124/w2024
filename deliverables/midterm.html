# Joint Project

The final project will be a two part joint assignment between MAD9124 and MAD9022. You will build a complete full-stack solution. The web service API portion will be graded for MAD9124, and the front-end application functionality, design, and usability will be graded for MAD9022.

# The Brief

Using Node.js, Express, and Redis, you will build the web service for a prototype of a fictitious application integrated with the MovieD (opens new window). This initial proof of concept (PoC) will have limited capabilities but should demonstrate the potential of a full-scale application.

The Movie Database documentation can be found here (opens new window)

# Core Requirements

  1. Using the Express.js framework, the API will follow these user stories:

  • A user can do a text search for movies that returns a limited response, designed to populated a search page.

  • A user can sort this response either by popularity, release date, or by votes.

  • A user can lookup a movie by id to get a more detailed response.

  • Each detailed response returned by id will be cached in Redis to save requests to the MovieDb.

  • A user can get metadata about image baseUrl and available sizes.

  1. The routes will be as follows:

  • The 3 text search routes:

    • /api/popularity?keyword=[text search]: The searched for movies sorted by popularity

    • /api/release-date?keyword=[text search]: The searched for movies sorted by release date

    • /api/vote?keyword=[text search]: The searched for movies sorted by votes

    • Each of these should have the following format:

    {
  "id": "Number",
  "title": "String",
  "poster_path": "String"
}

  • /api/id/:id: The detailed response. Covered by the Redis cache.

    • Should return the full response from the MovieDB.

    • Each request should lookup the id in the Redis cache, and not make a new request to the MovieDb if found.

    • If it's the first time seeing the id, it should save the response from the MovieDB in the Redis cache.

  • /api/meta: The metadata for the application:

    • Should follow the format:

      {
  "meta": {
    "images": {
      "baseUrl": "[base url]",
      "sizes": [
        /*array of image sizes*/
      ]
    }
  }
}
  1. Hosting
  • Your server needs to be hosted on the cloud. Using Render (opens new window), create a Web Service linked to your midterm repository, add the required Environment Variables, and publish your api to the cloud!
  1. Ensure that you write clean and readable code. Pay attention to:
  • No runtime errors
  • Consistent 2 space indentation
  • Logical grouping of related code
  • Semantically descriptive names for variables and functions
  • Well organized project folder structure
  • Properly formatted package.json file
    • Correct project name
    • Your author details
  • Properly written package.json scripts (dev + start)

CORS

We didn't have time to talk about CORS in class yet, but please follow these instructions to stop your server from sending CORS errors!

  1. Install cors
   npm i cors
  1. Import cors in src/index.js
const cors = require("cors");
  1. Add the following line to you code, after defining your app:
const app = express();
// add the following line:
app.use(cors());
